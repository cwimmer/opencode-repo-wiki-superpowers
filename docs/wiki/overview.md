# Overview

`repo-wiki-superpowers` is an **OpenCode plugin** that integrates the
**Karpathy LLM Wiki** documentation workflow with the **obra/superpowers**
development framework. It ships exactly two things and nothing else executable:

1. `llm-wiki` — build, query, and audit a repo-local Karpathy-style wiki under
   `docs/wiki/`. Each wiki page is generated by an LLM and cites the real
   source files it was built from ([repo-map](repo-map.md), [architecture](architecture.md)).
2. `wiki-context` — make a Superpowers **brainstorming/design** session use
   that wiki as memory, *grounded back to source* (source and tests win;
   [open-questions](open-questions.md)).

Guiding principle (from the design spec):
> The wiki gives Brainstorming memory; the source-map rule keeps it honest.

The plugin is **skill-only**: it exposes the two skills via OpenCode's native
`skill` tool by appending `skills/` to `config.skills.paths` from a `config`
hook. There are no custom tools, no message injection, and no per-session
bootstrap ([architecture](architecture.md)).

## Why

Per the design spec (§1, §2), the goal is to ship a small, loadable OpenCode
plugin — installable via HTTPS Git URL, safe inside isolated devcontainers,
usable across ≥3 repos — that closes the loop between "code", "wiki", and
"brainstorming".

## Non-goals (from §3)

- No documentation site generator.
- No syncing service or daemon.
- No custom OpenCode tools, no per-session bootstrap injection.
- No private-GitHub-SSH default install path.
- No changes to obra/superpowers itself (compose, do not fork).
- No agents/commands or workflow automation beyond the two skills.

## Install (short version)

Add both plugins to a consuming repo's `opencode.json`, pinned to a tag
([operations](operations.md)):

```jsonc
{
  "plugin": [
    "superpowers@git+https://github.com/obra/superpowers.git",
    "repo-wiki-superpowers@git+https://github.com/cwimmer/opencode-repo-wiki-superpowers.git#v0.1.0"
  ]
}
```

Restart OpenCode. Both skills then appear in the native `skill` tool
([integrations](integrations.md)).
